## Warnings

1. **[W1] Haiku 3 deprecation en skill-executor** — `tools/telegram/skill-executor.js` línea 261 mapea `'haiku'` a `'claude-3-5-haiku-20241022'`. Según Research Agent (finding #2, impacto high): `claude-3-haiku-20240307` se depreca el 19 de abril 2026. Aunque el mapping actual usa 3-5 Haiku, hay riesgo de confusión si el nombre simbólico "haiku" no se documenta explícitamente como Haiku 4.5. **Recomendación:** Verificar que el alias `haiku` apunta a `claude-haiku-4-5` y documentarlo en SKILLS.md y RULES.md.

2. **[W2] TOOLS.md desactualizado: 40+ tools no documentados** — El real conteo de `.js` en `tools/` es **86 archivos**, pero TOOLS.md declara "46 tools" (last updated 2026-03-13). Directorios enteros no documentados: `tools/images/` (4 tools: generate-image, generate-service, track-generation, cli), `tools/core/` (5 tools: tool-executor, context-builder, event-logger, validate-tool-execution, test-*), `tools/telegram/` extendido (11+ tools: agent-router, crud-handler, crud-parsers, crud-formatters, crud-permissions, audit-logger, auth-middleware, rate-limiter, input-sanitizer, adapters/*), y `tools/db/` expandido (conversation_queries, telegram_user_queries, update_project_status, run-migrations, etc.). **Recomendación:** Actualizar TOOLS.md con todas las nuevas categorías.

3. **[W3] data-agent.md falta `wat-memory.js` en tools** — La regla core del sistema WAT exige que todo agente liste `tools/db/wat-memory.js`. El `data-agent.md` omite esta tool en su sección "Tools disponibles". Solo lista `memory.js`, pero no `wat-memory.js`. **Recomendación:** Añadir `tools/db/wat-memory.js` a la lista de tools del Data Agent.

4. **[W4] dev-agent.md falta `wat-memory.js` en tools** — Mismo problema que W3. El `dev-agent.md` no lista `tools/db/wat-memory.js`. **Recomendación:** Añadir `tools/db/wat-memory.js` a la lista de tools del Dev Agent.

5. **[W5] marketing-agent.md lista skills incorrectamente asignadas** — El archivo `marketing-agent.md` lista bajo "Skills disponibles" a `/mapa-viaje-cliente` y `/estrategia-gtm`. Sin embargo, en SKILLS.md ambas están asignadas al `[[pm-agent]]`, no al `[[marketing-agent]]`. Esto genera confusión sobre qué agente debe invocarlas. **Recomendación:** Corregir marketing-agent.md para eliminar esas dos skills de su lista, o actualizar SKILLS.md si la asignación intencional es al marketing-agent.

6. **[W6] TOOLS.md declara estadísticas de uso desactualizadas** — La sección "Estadísticas de Uso (últimas 30d)" está claramente desactualizada (data from 2026-03-13). Los contadores reales de WAT Memory muestran `skill_invocation_count` de solo 3 (data-agent) y 4 (dev-agent), inconsistente con "1,247 invocaciones de memory.js". **Recomendación:** Eliminar o marcar como "no actualizable manualmente" esa sección, o reemplazarla con un comando dinámico.

7. **[W7] Opus 4.6 defaults a medium effort sin documentar** — Research Agent (finding #8, impacto high): Opus 4.6 ahora usa medium effort por defecto. Skills estratégicas de Emiralia (`/crear-prd`, `/estrategia-producto`, `/estrategia-gtm`, `/ui-ux-pro-max`) usan `model: opus`. No hay ninguna instrucción en ningún SKILL.md sobre usar `ultrathink` para forzar high effort cuando se necesita. **Recomendación:** Añadir nota en los SKILL.md de skills `opus` críticas indicando usar `ultrathink` si se requiere análisis profundo.

8. **[W8] AGENTS.md menciona "dev-agent.md" en ruta incorrecta** — AGENTS.md no tiene una entrada para el Dev Agent en la tabla principal de agentes. El archivo existe en `.claude/agents/dev/dev-agent.md` pero el Dev Agent no aparece listado en las tablas de AGENTS.md (ni en Internos, ni Mixtos, ni Operaciones). Solo aparece en la "Matriz de Coordinación". **Recomendación:** Añadir Dev Agent a la tabla de Internos en AGENTS.md.

9. **[W9] SKILLS.md dice "35+ skills" pero hay 41 SKILL.md** — SKILLS.md y CLAUDE.md declaran "35+ skills" pero el inventario real muestra **41 SKILL.md** en el sistema. El conteo en la Quick Reference de CLAUDE.md también está desactualizado. **Recomendación:** Actualizar referencias numéricas a "41+ skills".

10. **[W10] Auto-memory de Claude Code no integrada con WAT Memory** — Research Agent (finding #10, impacto high, score 36): Auto-memory feature lanzada en v2.1.59. La memoria auto de Claude Code y WAT Memory son dos sistemas paralelos sin integración documentada. Existe riesgo de datos en auto-memory que contradicen wat-memory.js. **Recomendación:** Documentar cómo coexisten ambos sistemas en RULES.md o crear guía de uso.

11. **[W11] `research-monitor` skill usa `model: haiku`** — `.claude/skills/ops/research-monitor/SKILL.md` tiene `model: haiku`.
